# Walkthrough: Evernote Archive Navigator & MDBrowser Integration

로컬 에버노트 마크다운 뷰어와 중복 파일 정리기에 이어, 본문이 정상적으로 렌더링되지 않는 예외 상황에 대처할 수 있도록 **3단 뷰어 옵션 전환 토글(Preview / MD Source / HTML Source)** 기능을 수려하게 추가 연동 완료하였습니다!

---

## 🛠️ What We Accomplished

### 1. 3-Pane 중복 파일 정리 전용 UI 추가 (`templates/cleaner.html` 연동)
* **좌측 Pane (Notebooks)**: 노트북 디렉토리 트리 구조를 한 눈에 파악할 수 있는 탐색기입니다.
* **가운데 Pane (남겨둘 원본 파일)**: 접미사가 없는 원본 파일(예: `Note.md`) 또는 중복 파일 중 가장 최우선순위(번호가 가장 작은 파일) 파일 1개를 안전하게 보호 대상(Keep)으로 자동 분류하여 보여줍니다.
* **우측 Pane (지울 중복 파일 목록)**: 정리해야 할 `*_1.md`, `*_2.md` 중복 파일들의 목록과 개별/전체 선택 체크박스, 그리고 하단에 삭제를 수행하는 붉은색 **"🔥 선택한 중복 파일 지우기" Action Button**이 배치되어 있습니다.

### 2. 뷰어 옵션 세그먼트 컨트롤러 추가 (`templates/index.html` 수정)
* **Preview (기본 렌더링)**: 기존 마크다운 본문 파싱 뷰를 그대로 유지합니다.
* **MD Source (마크다운 소스)**: 노트북의 마크다운 텍스트 원문(YAML Front Matter 제외)을 줄바꿈과 띄어쓰기가 완벽히 유지된 코드 포맷으로 보여줍니다.
* **HTML (컴파일 HTML 소스)**: `marked.js`에 의해 최종 파싱된 결과물인 순수 HTML 태그 코드를 직접 확인할 수 있습니다.
* **디자인 & 인터랙션**: 
  - 메타데이터 및 태그 패널 우측 옆자리에 **유리 모핑(Glassmorphic) 세그먼트 컨트롤 버튼바**를 탑재하여 전체 UI와 자연스럽게 어우러지도록 설계했습니다.
  - 마우스 오버 시 투명도 가변 효과와 부드러운 액티브 컬러 피드백 트랜지션을 연동했습니다.

### 3. 고품격 소스 코드 뷰어 및 구문 강조 연동 (Prism.js 통합)
* 단순히 플레인 텍스트로 원문을 보여주면 가독성이 현저히 떨어지므로, `MD Source` 및 `HTML` 모드로 전환 시 기존 뷰어에 장착된 **Prism.js 구문 강조기(Syntax Highlighting)**를 동적으로 바인딩 처리했습니다.
* 마크다운 문법과 HTML 태그 문법 각각에 맞추어 알록달록하고 극도로 선명한 가독성 테마(Prism Tomorrow Theme)가 실시간으로 재적용됩니다.
* 코드의 브라우저 비정상 동작을 막기 위해 태그 원문을 안전하게 이스케이프(`escapeHtml`) 처리한 후 주입하는 안전 장치를 마련했습니다.

### 4. 노트북 및 메모리 캐싱 최적화
* API 네트워크 트래픽을 아끼고 스위칭 속도를 눈부시게 끌어올리기 위해, 노트를 한 번 불러올 때 마크다운 원본과 파싱된 HTML을 각각 메모리 글로벌 캐시 변수(`currentNoteRawMarkdown`, `currentNoteRenderedHtml`)에 세팅해 둡니다.
* 새로운 노트를 선택하여 읽기 시작하면 뷰 모드가 자동으로 **`Preview` (기본값)** 모드로 부드럽게 초기화되어 실생활 사용성을 극대화했습니다.

### 5. 개별 노트 삭제(Delete) 및 리얼타임 동기화 기능 추가 [NEW]
* **🗑️ 개별 삭제(Delete) 버튼 탑재**: 뷰어 헤더 영역의 뷰 모드 세그먼트 바로 우측에 붉은색 HSL 그라데이션 광채의 `🗑️ Delete` 버튼을 깔끔하게 추가했습니다.
* **🔒 경로 탐색 차단 보안 백엔드 API (`/api/notes/delete` POST)**: 디렉토리 상위 탐색(`..`, `/`, `\\`)을 막고 오직 `.md` 마크다운 파일만 삭제할 수 있도록 엄격한 경로 유효성 검증 로직이 포함된 보안 API를 신설하였습니다.
* **✨ 미려한 초록색 토스트(Toast) 팝업**: 삭제 성공 시 화면 우측 하단에서 부드러운 3D 가속 애니메이션을 타며 슬라이드 인-아웃되는 초록색 토스트 팝업 창을 마크업 및 CSS 키프레임 애니메이션으로 완전히 보완 설계했습니다.
* **⚡ 실시간 비동기 UI 리프레시**: 노트가 성공적으로 삭제되면 웹페이지가 전체 새로고침되는 불편함 없이, 좌측 탐색기의 노트북별 노트 개수, 태그 카운트 배치, 가운데 카드 리스트가 비동기(`fetch`)로 즉시 갱신되어 삭제된 항목이 화면에서 깔끔히 사라집니다. 뷰어 영역 또한 즉시 Empty State로 매끄럽게 회귀합니다.

### 6. 코드 리팩토링 및 스타일시트 공용화 (Shared CSS) [NEW]
* **🎨 공용 스타일시트 통합 분리 (`static/style.css` 신설)**:
  - `index.html`과 `cleaner.html`에 중복 탑재되어 불필요하게 템플릿 용량을 차지하던 800여 줄의 인라인 CSS 코드를 단 하나의 프리미엄 공용 스타일시트(`static/style.css`)로 완벽하게 통합하였습니다.
  - 마크업 본문 크기가 획기적으로 경량화되어 가독성과 유지보수성이 비약적으로 증가했습니다.
* **🏷️ 레이아웃 바인딩 표준화**:
  - `body` 태그에 클래스 바인딩 방식(`.index-layout`, `.cleaner-layout`)을 채택하여, 배경 라디언트 광채 효과와 Flex 정렬 방향을 개별 템플릿 목적에 맞게 스마트하게 최적화했습니다.

## 🚀 How to Run and Test

1. **`C:\_My2026\_EVERBK\_MDBrower`** 폴더로 이동합니다.
2. **`run_server.bat`** 파일을 더블클릭하여 로컬 Flask 서버를 구동합니다.
3. 웹 브라우저에서 임의의 노트를 클릭하여 본문을 엽니다.
4. 본문 헤더 우측의 **Preview | MD Source | HTML** 세그먼트 버튼을 통해 뷰어 옵션 전환을 확인합니다.
5. 우측에 배치된 **🗑️ Delete** 버튼을 클릭하고 확인창에서 승인하여 리얼타임 동기화 삭제 조작을 테스트합니다.
6. 로컬 뷰어와 중복 파일 정리기(`cleaner`) UI가 동일하게 수려하고 선명한 어두운 유리 테마(Glassmorphism Dark UI)를 깨짐 없이 일관되게 렌더링하는지 감상하세요!
